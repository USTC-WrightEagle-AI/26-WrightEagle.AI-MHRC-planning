### 4. 在机器人上部署Sherpa-ONNX实现的语音交互系统
> **技术背景**: Sherpa-ONNX 基于下一代 Kaldi 和 ONNX Runtime，支持完全离线、跨平台部署，具备低延迟特性，适合嵌入式设备。

- [ ] **基础环境搭建**
    - [ ] 编译或安装 `sherpa-onnx` Python/C++ API。
    - [ ] 下载所需的ONNX模型（Transducer/Paraformer等）。
- [ ] **功能模块实现**
    - [ ] **流式ASR**: 实现实时语音转文字，模拟麦克风输入流。
        - [ ] *验收标准*: 延迟 < 300ms，支持边说边出字。
    - [ ] **非流式ASR + VAD**: 集成 Silero VAD 进行语音端点检测，实现整句识别。
        - [ ] *验收标准*: 准确识别起止点，无截断。
    - [ ] **非流式ASR + VAD + 动态说话人识别**: 集成说话人日志（Speaker Diarization）功能。
        - [ ] *验收标准*: 能区分不同说话人（"说话人A: ...", "说话人B: ..."）。
- [ ] **输出物**: `sherpa_onnx_demo/` 代码库，各功能模块测试视频/日志。

### 5. 部署Qwen3-ASR模型，并测试效果
> **技术背景**: Qwen3-ASR (1.7B/0.6B) 是阿里最新开源模型，支持52种语种/方言，在噪声环境和口音识别上表现优异 (SOTA)。

- [ ] **模型获取与加载**
    - [ ] 从 ModelScope 或 HuggingFace 下载 `Qwen3-ASR-1.7B` (高精度) 或 `0.6B` (轻量级)。
    - [ ] 配置推理环境（需较高显存，建议1.7B模型至少需要8GB+显存）。
- [ ] **功能测试**
    - [ ] **多语种/方言测试**: 使用包含中文方言、英语口音的测试集。
    - [ ] **抗噪测试**: 使用步骤2中收集的机器人噪声数据进行测试。
    - [ ] **特殊场景**: 测试带背景音乐或歌唱内容的识别能力。
- [ ] **输出物**: `qwen3_asr_evaluation_report.md` (包含WER/CER指标，与Faster-Whisper的对比)。
