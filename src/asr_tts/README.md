# ROS ASR & TTS Package

A ROS (Robot Operating System) package for **speech recognition** and **text-to-speech**, built on [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx). Provides **5 ASR modes** with a unified TTS node, all publishing/subscribing to the same ROS topics.

## Project Structure

```
workspace/src/
    asr_tts/                              # This package
        CMakeLists.txt
        package.xml
        README.md                          # This file
        launch/
            speech.launch                 # Generic launch (supports ASR switching)
            cade_voice.launch             # CADE robot launch (preset devices)
        scripts/
            # === TTS Node (always used) ===
            tts_node.py                   # TTS synthesis (VITS / Kitten)

            # === ASR Nodes (pick one) ===
            asr_node.py                   # [Default] VAD + Offline ASR (SenseVoice/Whisper)
            streaming_asr_endpoint_node.py # Streaming ASR + Endpoint Detection (Zipformer)
            vad_streaming_asr_node.py     # VAD + Streaming ASR (Zipformer, no speaker ID)
            dynamic_vad_asr_node.py       # VAD + Offline ASR + Dynamic Speaker Registration
            dynamic_vad_streaming_asr_node.py # VAD + Streaming ASR + Dynamic Speaker Registration
        models/                            # Model files (gitignored, download separately)
        recordings/                        # Audio recording output directory
        speakers/
            speaker_db.json                # Speaker database (auto-generated by dynamic nodes)
```

## Quick Start

### 1. Download Models

Place model files in `asr_tts/models/`:

| Model | Purpose | Required |
|-------|---------|----------|
| `silero_vad.onnx` | Voice Activity Detection | **Yes** |
| `sherpa-onnx-sense-voice-zh-en-ja-ko-yue-2024-07-17/` | Multilingual offline ASR (default) | **One of ASR** |
| `sherpa-onnx-whisper-medium.en/` | English Whisper offline ASR | Optional |
| `sherpa-onnx-streaming-zipformer-en-2023-06-26/` | English streaming ASR (Zipformer) | **Streaming nodes** |
| `wespeaker_en_voxceleb_CAM++_LM.onnx` | Speaker embedding model | **Dynamic speaker nodes** |
| `vits-zh-aishell3/` | Chinese TTS model (VITS) | **Yes** |

### 2. Install Dependencies

```bash
pip install -r requirements.txt
```

### 3. Build & Run

```bash
# Build
catkin_make

# Source environment
source devel/setup.bash

# Launch (default: asr_node with SenseVoice)
roslaunch asr_tts speech.launch
```

## ASR Node Switching Guide

This package provides **5 ASR modes**. Select one via the `asr_type` argument in `speech.launch`:

### Method 1: Command-line argument (recommended)

```bash
# Default: VAD + SenseVoice offline ASR
roslaunch asr_tts speech.launch

# Streaming ASR with endpoint detection (Zipformer)
roslaunch asr_tts speech.launch asr_type:=streaming_endpoint

# VAD + Streaming ASR (no speaker identification)
roslaunch asr_tts speech.launch asr_type:=vad_streaming

# Dynamic speaker registration + offline ASR
roslaunch asr_tts speech.launch asr_type:=dynamic_vad

# Dynamic speaker registration + streaming ASR
roslaunch asr_tts speech.launch asr_type:=dynamic_vad_streaming
```

### Method 2: Edit default in launch file

Edit `launch/speech.launch`, change:
```xml
<arg name="asr_type" default="asr_node" />
```

### CADE Robot Preset

```bash
# Uses cade_voice.launch with preset device names (CADE_Mic / CADE_Speaker)
roslaunch asr_tts cade_voice.launch
roslaunch asr_tts cade_voice.launch asr_type:=streaming_endpoint
```

## Node Descriptions

### Common Features (all upgraded nodes)

| Feature | Description |
|---------|-------------|
| **Device locking** | Precise microphone/speaker selection via `device_name` ROS param |
| **Callback-based recording** | PortAudio callback + queue producer-consumer pattern |
| **Auto resampling** | Adapts device sample rate to model rate (16 kHz) |
| **Unified topic** | All ASR nodes publish to `/asr` (`std_msgs/String`) |
| **Unified TTS** | TTS node subscribes to `/tts`, supports VITS/Kitten dual architecture |

---

### 1. `asr_node` -- VAD + Offline ASR (Default)

| Property | Value |
|----------|-------|
| **ASR Engine** | OfflineRecognizer (SenseVoice / Whisper / Paraformer / Transducer) |
| **VAD** | Silero VAD (segment-level detection) |
| **Speaker ID** | None (fixed `speaker0`) |
| **Use case** | Multilingual recognition, Chinese-primary scenarios |
| **Default model** | sherpa-onnx-sense-voice-zh-en-ja-ko-yue |

**Output**: `[speaker0]: hello world`

---

### 2. `streaming_asr_endpoint_node` -- Streaming ASR + Endpoint Detection

| Property | Value |
|----------|-------|
| **ASR Engine** | OnlineRecognizer (Zipformer Transducer) |
| **Endpoint Detection** | Built-in rule-based endpoint detection |
| **VAD** | None (continuous streaming input) |
| **Speaker ID** | None |
| **Use case** | Real-time English streaming transcription, low-latency |

**Output**: `[speaker0]: hello world` (partial results + FINAL tag)

---

### 3. `vad_streaming_asr_node` -- VAD + Streaming ASR

| Property | Value |
|----------|-------|
| **ASR Engine** | OnlineRecognizer (Zipformer Transducer) |
| **VAD** | Silero VAD (detects speech boundaries) |
| **Endpoint Detection** | Yes (rule-based) |
| **Speaker ID** | None |
| **Use case** | English streaming with silence detection |

**Output**: `[Speech detected]` -> `[0] [speaker0]: text`

---

### 4. `dynamic_vad_asr_node` -- Dynamic Speaker + VAD + Offline ASR

| Property | Value |
|----------|-------|
| **ASR Engine** | OfflineRecognizer (Whisper/SenseVoice/etc.) |
| **VAD** | Silero VAD |
| **Speaker ID** | Yes -- Wespeaker embedding + cosine similarity matching |
| **Auto-registration** | Unknown speakers registered as `speaker0`, `speaker1`, ... |
| **Database persistence** | JSON file (`speakers/speaker_db.json`) |
| **Use case** | Multi-speaker conversations, need to distinguish speakers |

**Output**: `[NEW SPEAKER] Registered: speaker1` -> `[speaker1]: hello`

---

### 5. `dynamic_vad_streaming_asr_node` -- Dynamic Speaker + VAD + Streaming ASR

| Property | Value |
|----------|-------|
| **ASR Engine** | OnlineRecognizer (Zipformer Transducer) |
| **VAD** | Silero VAD |
| **Speaker ID** | Yes -- Wespeaker embedding + cosine similarity matching |
| **Auto-registration** | Auto-registers unknown speakers |
| **Use case** | Multi-speaker + real-time streaming output (English) |

**Output**: `[Speech detected]` -> `[0] [speaker1]: hello`

---

### 6. `tts_node` -- Text-to-Synthesis (Universal)

| Property | Value |
|----------|-------|
| **TTS Architecture** | VITS (Chinese) / Kitten (English), auto-detected from path |
| **Playback method** | paplay (system player) > sounddevice (fallback) |
| **Resampling** | Auto-adapts to device sample rate |
| **Subscription** | `/tts` (`std_msgs/String`) |

---

## Topic Interface

| Topic | Type | Direction | Description |
|-------|------|-----------|-------------|
| `/asr` | std_msgs/String | Publish (all ASR nodes) | Recognized text result |
| `/tts` | std_msgs/String | Subscribe (TTS node) | Text to synthesize |

## ROS Parameters

| Parameter | Node(s) | Default | Description |
|-----------|---------|---------|-------------|
| `device_name` | All | `"default"` | Audio device name (substring match) |
| `mic_id` | launch | `"default"` | Microphone device name |
| `speaker_id` | launch | `"default"` | Speaker device name |
| `asr_type` | launch | `"asr_node"` | ASR node type selector |
| `save_audio` | ASR nodes | `""` or package path | Folder to save recorded audio segments (disabled if empty) |

## Merging to Main Project

This package is designed to be merged into your main project at `main/src/asr_tts`.

### Steps

```bash
# 1. Copy this package into your main project's src directory
cp -r ros_speech/src/asr_tts main/src/

# 2. Rebuild in the main workspace
cd main
catkin_make

# 3. Run
source devel/setup.bash
roslaunch asr_tts speech.launch
```

### Post-Merge Checklist

1. **Model files**: Ensure `main/src/asr_tts/models/` contains all required models
2. **CMakeLists.txt**: Already configured to install all 6 scripts; no changes needed
3. **Launch files**: Use directly; `asr_type` argument controls which ASR node starts
4. **CADE robot**: Use `cade_voice.launch`; default devices are `CADE_Mic` / `CADE_Speaker`
